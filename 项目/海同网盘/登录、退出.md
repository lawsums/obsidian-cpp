## **为什么 `deleteLater()` 解决了问题：**

### **`delete` vs `deleteLater()` 的区别：**

|特性|`delete`|`deleteLater()`|
|---|---|---|
|**执行时机**|立即删除|下次事件循环时删除|
|**线程安全**|不安全|线程安全|
|**信号槽**|可能导致已删除对象收到信号|安全，对象在当前事件循环中仍然有效|
|**Qt推荐**|不推荐用于QObject派生类|推荐用于QObject派生类|

### **你的代码问题分析：**

原来的代码：
```cpp
delete *iter;  // 立即删除
// 此时对象已被销毁，但可能还有：
// 1. 未处理的事件
// 2. 待发射的信号
// 3. 其他线程可能还在访问
```

修复后的代码：
```cpp
(*iter)->deleteLater();  // 延迟删除
// 对象在当前事件循环中仍然有效
// 所有待处理的事件都会处理完毕
// 然后在下一次事件循环时安全删除
```
