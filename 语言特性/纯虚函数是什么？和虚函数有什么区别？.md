**简单来说**：

- **虚函数**： “你可以重写我，但如果你不重写，我会提供一个默认的行为。”
- **纯虚函数**： “你**必须**重写我，我在这里不提供任何行为，只是告诉你这个功能是必需的。”

纯虚函数和虚函数都是C++中实现多态性的重要机制，它们都使用`virtual`关键字来声明，但两者在语法、用途和对类的影响上有显著区别。

### 1. 虚函数 (Virtual Function)

- **定义**：在基类中使用`virtual`关键字声明的函数。它可以在派生类中被**重写（override）**。
- **语法**：`virtual 返回类型 函数名(参数列表);`
- **实现**：虚函数在基类中**必须提供函数体**（即有具体的实现）。
- **目的**：允许派生类根据需要**选择性地重写**该函数，以提供不同的行为。如果派生类不重写，则会继承并使用基类的实现。
- **类的性质**：包含虚函数的类是**普通类**，可以被实例化（可以创建该类的对象）。

**示例**：

```cpp
class Animal {
public:
    virtual void makeSound() {
        cout << "Some generic animal sound" << endl;
    }
};

class Dog : public Animal {
public:
    void makeSound() override { // 重写虚函数
        cout << "Woof!" << endl;
    }
};
```

在这个例子中，`Animal`类可以被实例化，其`makeSound()`函数有默认实现。

### 2. 纯虚函数 (Pure Virtual Function)

- **定义**：一种特殊的虚函数，它在基类中**只声明，没有实现**，并且**强制要求**所有派生类必须提供自己的实现。
- **语法**：`virtual 返回类型 函数名(参数列表) = 0;`
- **实现**：纯虚函数在基类中**不能有函数体**。 `= 0` 表示这是一个纯虚函数。
- **目的**：为派生类定义一个**接口规范**或**契约**。它表示“这个函数必须被实现”，但不提供任何默认行为。常用于设计抽象基类。
- **类的性质**：包含纯虚函数的类被称为**抽象类 (Abstract Class)**。**抽象类不能被实例化**，即不能创建该类的对象。它只能作为基类被继承。

**示例**：

```cpp
class Shape {
public:
    virtual double area() = 0; // 纯虚函数
    virtual ~Shape() = default; // 虚析构函数是好习惯
};

class Circle : public Shape {
private:
    double radius;
public:
    Circle(double r) : radius(r) {}
    double area() override { // 必须重写纯虚函数
        return 3.14159 * radius * radius;
    }
};

class Rectangle : public Shape {
private:
    double width, height;
public:
    Rectangle(double w, double h) : width(w), height(h) {}
    double area() override { // 必须重写纯虚函数
        return width * height;
    }
};
```

在这个例子中：

- `Shape` 是一个抽象类，因为它包含纯虚函数 `area()`。
- 你**不能**写 `Shape s;`，这会导致编译错误。
- `Circle` 和 `Rectangle` 必须实现 `area()` 函数才能成为具体类并被实例化。

### 主要区别总结

|特性|虚函数 (Virtual Function)|纯虚函数 (Pure Virtual Function)|
|:--|:--|:--|
|**关键字**|`virtual`|`virtual` + `= 0`|
|**实现**|**必须**在基类中提供实现|**不能**在基类中提供实现|
|**重写**|派生类**可以**选择重写|派生类**必须**重写|
|**类的性质**|基类是**具体类**，**可以**实例化|基类是**抽象类**，**不能**实例化|
|**用途**|提供可选的多态行为，有默认实现|定义接口规范，强制派生类实现，无默认行为|
